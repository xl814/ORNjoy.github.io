Python 嵌套函数，闭包，修饰器

为了明白这三个概念，我试图从这个思路去理解：

首先，什么是嵌套函数，很简单就是一个函数中定义另一个函数，这个内部的函数有可能会用到外部函数的一下变量。那如果同时这个内部函数会返回一个自身的对象：

这就是一个闭包特性，准确来说这个inner()就被称为闭包函数，他要满足：

1.嵌套函数

2.需要引用外部函数的变量

3.需要返回一个对象（没有括号相当于一个引用）

有什么用呢？

正常来说，一个python函数的执行应该是当outer()结束完了后，其保存在栈里的局部变量name应该是没了，可是为什么我们调用s()的时候依然好使呢，这就是闭包的特性，准确来说这就是python对于一个对象的管理是按引用的，当s = out()实际是将inner函数对象的地址（或者说引用）复制隔了s,因此当我们基本重新运行out()修改其name值时，那是不影响s()的name的值的。这就是闭包的作用，保持程序上一次运行后的状态继续执行

进一步说，python中修饰器的原理也就是这个了。我们知道设计原则中有一个开发封闭原则：就是在不修改原来代码的基础上进行功能的扩展，不能改变其源代码，调用方式也不能改变，这样，是否和上述的闭包特性贴合了。比如当我们想要输出一个time的时候

这就是装饰器，通过闭包特性得到的是一个函数的引用，有了这个引用我们可以当作要扩展功能函数的参数，但为了不修改其函数调用方式在赋值给这个引用，当然这个修饰器还有更复杂，厉害的应用，我也没有太深入了解，目前就先这样吧。
